\todo{Later values are just functions, and thus if more than one depend on that Later, then the Later will be evaluated more than once. This can cause the heap to grow by N - 1 where N is the number of dependencies on the Later value. For Later signals this can be resolved by giving the Later an empty head and using the tail constructor to get the tail. In the evaluation of the signal heaps behaviour, we mention that the signals $[4]$ and $[6]$ are caused by a Later value being evaluated 2 times, once for each dependence on the Later. An example of how to fix this issue would be by using the tail constructor, to cache the result of the Later evaluation, but there is one for Later signals.
